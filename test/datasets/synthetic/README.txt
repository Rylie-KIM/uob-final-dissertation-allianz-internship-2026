
SYNTHETIC INSURANCE CLAIMS — COLUMN DICTIONARY
===============================================
Generated by: datasets/generate_synthetic.py
DGP: logit(P(fraud)) = -2.5 + 0.8*high_amount + 0.6*night_claim
                             + 1.2*high_postcode + 0.4*prior_claims + N(0, 0.3)

───────── BASE CLAIM FEATURES ─────────────────────────────────────────────────
claim_id          str     Unique claim identifier (CLM000001 format)
claim_amount      float   Raw claim amount in GBP (log-normal, mean ~exp(7.5))
claim_hour        int     Hour of claim submission (0–23)
postcode_risk     float   Postcode-level fraud risk index (0=low, 1=high)
prior_claims      int     Number of prior claims by same claimant (0–5)
claim_type        str     Claim type: theft | damage | injury

───────── DERIVED BINARY FEATURES (used as model inputs) ──────────────────────
high_amount       int     1 if claim_amount > 75th percentile, else 0
night_claim       int     1 if claim_hour in {22,23,0,1,2,3,4,5}, else 0
high_postcode     int     1 if postcode_risk > 0.6, else 0

───────── GROUND TRUTH (oracle — unknown in production) ───────────────────────
true_fraud        int     1 if claim is genuinely fraudulent, else 0
true_fraud_prob   float   True underlying fraud probability

───────── MODEL VERSION SCORES (versions 1, 2, 3) ─────────────────────────────
model_v{N}_score        float  Predicted fraud probability from model version N
model_v{N}_investigated int    1 if investigated under model vN's policy, else 0
model_v{N}_observed_fraud float 1/0 if investigated and fraud found/not found
                                NaN if uninvestigated (label is MISSING)

───────── CONVENIENCE COLUMNS (added in claims_with_versions.csv) ─────────────
investigated      int     = model_v3_investigated (current model's decision)
observed_fraud    float   = model_v3_observed_fraud (NaN for uninvestigated)

───────── LOOP METRICS (loop_metrics.csv) ──────────────────────────────────────
version                     int    Model version number (1, 2, 3, ...)
n_investigated              int    Number of claims investigated in that generation
investigation_rate          float  Fraction of all claims investigated
observed_fraud_rate         float  Fraud rate WITHIN investigated claims (BIASED)
auc_on_investigated         float  AUC computed on investigated test set (BIASED)
true_recall                 float  Recall vs ground truth (ALL claims)
precision                   float  Precision on investigated claims
loop_amplification_factor   float  fraud_rate_top_q / true_fraud_rate (>1 = loop active)
blind_spot_fraud_fraction   float  Fraction of true fraud in NEVER-investigated claims

KEY RELATIONSHIPS:
  observed_fraud_rate >> true_fraud_rate    ← investigation bias inflates observed rate
  auc_on_investigated >> true_auc           ← biased evaluation looks better than reality
  true_recall decreases over versions       ← loop creates growing blind spots
  loop_amplification_factor increases       ← model gets better at finding where it looks,
                                              worse at finding fraud it doesn't look for
