## 🇰🇷 한국어 요약

### 1. 문제 정의 — "Selective Labels Problem"

머신러닝 모델의 성능을 평가할 때, **라벨 자체가 과거 인간 의사결정자의 선택의 결과물**인 경우가 많다.

- **예시 (사법):** 판사가 보석을 허가한 피고인에게만 "재범 여부" 라벨이 존재. 구속된 피고인은 라벨 자체가 없음 (NA).
- **예시 (의료):** 의사가 처방한 환자에게만 "재발 여부" 라벨이 존재.
- **예시 (보험 — Allianz와 직결):** 보험 관리자가 승인한 청구건에 대해서만 "수익/손실" 결과를 관측. 거절된 청구건은 라벨 없음.

이 데이터로 학습/평가하면 **편향된 성능 추정**을 얻게 된다. 모델이 "본 적 있는" 영역만 평가되기 때문.

### 2. 두 가지 기술적 어려움

논문이 명시하는 핵심 두 요소:

1. **Selective labels** — 인간 의사결정자가 어떤 인스턴스에 라벨이 붙는지 결정함 (yes → label 존재, no → NA)
2. **Unobservables (Z)** — 인간은 알지만 알고리즘은 모르는 변수가 존재. 예: 판사가 법정에서 본 피고인의 태도, 의사가 진료실에서 본 환자의 표정.
   - Z는 **결정(T)에도 영향, 결과(Y)에도 영향** → 측정되지 않은 confounder

이 두 가지가 **동시에** 존재하면, 기존의 counterfactual inference 기법(IPW, doubly robust, propensity matching 등)은 "no unmeasured confounders" 가정을 위배해 **사용 불가**.

### 3. 제안: Contraction Technique

핵심 아이디어: **여러 의사결정자의 이질성(heterogeneity)을 자연 실험처럼 활용**한다.

#### 3-1. 세 가지 전제 조건

| 조건 | 의미 |
|------|------|
| (i) Multiple decision-makers | 의사결정자가 한 명이 아니라 여럿 존재 |
| (ii) Random assignment | 인스턴스가 의사결정자에게 무작위 배정됨 (또는 비슷한 분포) |
| (iii) Heterogeneity in acceptance rates | 의사결정자마다 "yes" 비율(acceptance rate)이 다름 — 즉 관대한 판사와 엄격한 판사가 공존 |

#### 3-2. 알고리즘 (Algorithm 1)

```
Input: 관측 데이터 D, 모델 점수 S, 목표 acceptance rate r
1. q = D 안에서 acceptance rate가 가장 높은 의사결정자 (가장 관대한 판사)
2. D_q = q가 본 모든 케이스
3. R_q = D_q 중 라벨이 관측된 케이스 (q가 yes 결정한 것들)
4. R_q를 모델 점수 S 기준 내림차순 정렬 (위험 높은 순)
5. R_q의 상위 ⌊(1-r)|D_q| - (|D_q| - |R_q|)⌋ 개를 제거
   → 즉 q가 풀어줬지만 모델이 "위험"이라 본 사람들을 빼버림
6. 남은 케이스 R_B로 failure rate u 계산
7. Return u
```

#### 3-3. 직관적 설명

가장 관대한 판사 q (acceptance 90%)와 더 엄격한 판사 k (acceptance 80%)가 있다고 하자.
- q가 1000명 중 900명 풀어줌 → 900명의 라벨 관측 가능
- 모델이 q가 풀어준 900명 중 "위험도 높음" 100명을 골라 다시 가둠 → 800명 남음
- 이제 모델의 hybrid acceptance rate = 80% (k와 동일)
- 800명의 failure rate를 k의 failure rate와 비교 → **공정한 비교 가능**

핵심: **imputation을 전혀 하지 않음.** 라벨이 없는 영역은 그냥 건드리지 않는다.

#### 3-4. 오차 상한 (Proposition 4.1)

Contraction의 실패율 추정치 u와 진짜 값의 차이는 다음을 절대 넘지 않는다:

$$\text{Error} \leq \frac{(1-a) \cdot |D_q - R_q|}{|D_q|}$$

여기서 a는 B와 q가 "no decision"에 합의하는 비율. **가장 관대한 판사의 acceptance rate가 높을수록**, **B와 q의 일치율이 높을수록** 오차가 작아진다.

## 🇬🇧 English Summary

### 1. Problem — The "Selective Labels Problem"

When evaluating ML models, **labels themselves are often consequences of past human decisions**:

- **Judicial bail:** outcome ("crime / no crime") only observed for defendants the judge released.
- **Medical:** outcome ("relapse") only observed for patients the doctor treated.
- **Insurance (directly relevant to Allianz):** profit/loss outcome only observed for claims/applications the manager approved.

Training/evaluating on this data yields **biased performance estimates** because the model is judged only on regions humans "saw fit" to expose.

### 2. Two Technical Obstacles

The paper isolates **two simultaneous ingredients** that break standard evaluation:

1. **Selective labels** — humans determine which instances get labels (yes → label, no → NA).
2. **Unobservables (Z)** — variables visible to humans but not to the algorithm (e.g., a defendant's demeanour in court, a patient's appearance in clinic). Z influences **both** the decision T **and** the outcome Y → it is an **unmeasured confounder**.

When both obtain, **counterfactual inference techniques fail** because they require the "no unmeasured confounders" assumption.

### 3. Proposed Method — Contraction

Core idea: **exploit heterogeneity across decision-makers as a natural experiment.**

#### 3-1. Three Required Conditions

| Condition | Meaning |
|-----------|---------|
| (i) Multiple decision-makers | More than one human judge / doctor / manager |
| (ii) Random assignment | Subjects assigned to decision-makers (approximately) at random |
| (iii) Heterogeneous acceptance rates | Decision-makers differ in how lenient they are |

#### 3-2. Algorithm (Algorithm 1)

```
Input: Observational data D, model risk scores S, target acceptance rate r
1. q ← most lenient decision-maker in D (highest acceptance rate)
2. D_q ← all cases seen by q
3. R_q ← cases in D_q for which labels exist (q said yes)
4. Sort R_q by S in descending order (highest model risk first)
5. Remove top ⌊(1-r)|D_q| - (|D_q| - |R_q|)⌋ rows
   (i.e., have the model "detain" those q released but the model rates risky)
6. Compute failure rate u over the remaining set R_B
7. Return u
```

#### 3-3. Intuition

Suppose the most lenient judge q has acceptance rate 90%, and a stricter judge k has 80%.
- q released 900 of 1000 → labels exist for those 900.
- The model contracts q's set by re-detaining the 100 highest-risk releases.
- Hybrid model+q acceptance rate = 80%, matching k.
- Compare failure rates on these 800 vs. k's failure rate → **a fair comparison at equal acceptance rate**.

The key feature: **no imputation is performed.** Regions with missing labels are simply left alone.

#### 3-4. Worst-Case Error Bound (Proposition 4.1)

$$\text{Error} \leq \frac{(1-a) \cdot |D_q - R_q|}{|D_q|}$$

where *a* is the fraction of "no decisions" on which B and q agree. **Higher acceptance rate of the most lenient judge** and **higher B–q agreement** tighten the bound.

### 4. Experimental Results

#### 4-1. Synthetic Data
- M = 100 decision-makers, 500 cases each, N = 50k observations.
- Features: **X (visible to both), Z (visible only to humans), W (visible to neither)**.
- Logistic regression as the predictive model.

**Figure 3 — headline:**
- True Evaluation (green) ≈ Contraction (blue) — near perfect overlap.
- Imputation methods (gradient boosting, logistic regression, nearest neighbour, propensity score matching, doubly robust) **severely underestimate** failure rate → falsely make the model look better than humans.

**Figure 4:** As Z's influence (β_Z) grows, imputation errors explode while contraction stays accurate (6.4× smaller MAE at β_Z = 1).

#### 4-2. Real-World Datasets

| Domain | Size | Decision-makers |
|--------|------|----------------|
| Bail decisions | ~9k defendants | 18 judges |
| Medical (asthma) | ~60k patients | 39 doctors |
| **Insurance approvals** | **~50k requests** | **74 managers** |

**Figure 6 — headline:**
- "Labeled outcomes only" evaluation suggests the model beats humans → **overoptimistic**.
- Contraction reveals the model actually **underperforms** human decision-makers, because humans use unobservable Z.

### 6. Limitations (acknowledged in the paper)

- Requires **random assignment** of subjects to decision-makers (validated via F-test).
- If the **most lenient decision-maker's acceptance rate is low**, the error bound widens.
- Inapplicable to **single-decision-maker** settings (e.g., a single fraud team) — for Allianz, we should check whether treating model versions as decision-makers is defensible.
